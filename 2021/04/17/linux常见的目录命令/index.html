linux常见的目录命令

linux
技术
Publish Date:   2021-04-17

常用的基本命令（必掌握）

目录管理

绝对路径和相对路径

我们知道Linux的目录结构为树状结构，最顶级的目录为根目录 /。

其他目录通过挂载可以将它们添加到树中，通过解除挂载可以移除它们。

在开始本教程前我们需要先知道什么是绝对路径与相对路径。

绝对路径：

路径的写法，由根目录 / 写起，例如：/usr/share/doc 这个目录。

相对路径：

路径的写法，不是由 / 写起，例如由 /usr/share/doc 要到 /usr/share/man 底下时，可以写成：cd ../man 这就是相对路径的写法啦！

处理目录的常用命令

接下来我们就来看几个常见的处理目录的命令吧：

  • ls: 列出目录
  • cd：切换目录
  • pwd：显示目前的目录
  • mkdir：创建一个新的目录
  • rmdir：删除一个空的目录
  • cp: 复制文件或目录
  • rm: 移除文件或目录
  • mv: 移动文件与目录，或修改文件与目录的名称

你可以使用 man [命令] 来查看各个命令的使用文档，如 ：man cp。

ls （列出目录）

在Linux系统当中， ls 命令可能是最常被运行的。

语法：

[root@www ~]# ls [-aAdfFhilnrRSt] 目录名称

选项与参数：

  • -a ：全部的文件，连同隐藏文件( 开头为 . 的文件) 一起列出来(常用)
  • -l ：长数据串列出，包含文件的属性与权限等等数据；(常用)

将目录下的所有文件列出来(含属性与隐藏档)

[root@www ~]# ls -al ~

cd （切换目录）

cd是Change Directory的缩写，这是用来变换工作目录的命令。

语法：

cd [相对路径或绝对路径]

测试：

# 切换到用户目录下
[root@VM-0-17-centos /]# cd home
# 使用 mkdir 命令创建 tom 目录
[root@VM-0-17-centos home]# mkdir tom
[root@VM-0-17-centos home]# ls
kevintam  tom
# 进入  tom 目录
[root@VM-0-17-centos home]# cd tom/
[root@VM-0-17-centos tom]# 
# 回到上一级
[root@VM-0-17-centos tom]# cd ..
[root@VM-0-17-centos home]# 
# 回到根目录
[root@VM-0-17-centos home]# cd /
[root@VM-0-17-centos /]# 
# 表示回到自己的家目录，亦即是 /root 这个目录
[root@VM-0-17-centos /]# cd ~
[root@VM-0-17-centos ~]#

pwd ( 显示目前所在的目录 )

pwd 是 Print Working Directory 的缩写，也就是显示目前所在目录的命令。

[root@kuangshen kuangstudy]#pwd [-P]

选项与参数：**-P** ：显示出确实的路径，而非使用连接(link) 路径。

测试：

# 单纯显示出目前的工作目录
[root@VM-0-17-centos tom]# pwd
/home/tom
# 如果是链接，要显示真实地址，可以使用 -P参数
[root@VM-0-17-centos tom]# pwd -P
/home/tom

mkdir （创建新目录）

如果想要创建新的目录的话，那么就使用mkdir (make directory)吧。

mkdir [-mp] 目录名称

选项与参数：

  • -m ：配置文件的权限喔！直接配置，不需要看默认权限 (umask) 的脸色～
  • -p ：帮助你直接将所需要的目录(包含上一级目录)递归创建起来！

测试：

# 进入我们用户目录下
[root@VM-0-17-centos /]# cd home/
# 创建一个 test 文件夹
[root@VM-0-17-centos home]# mkdir test
[root@VM-0-17-centos home]# ls
kevintam  test  tom
# 创建多层级目录
[root@VM-0-17-centos home]# mkdir test1/test2
mkdir: cannot create directory ‘test1/test2’: 
No such file or directory# <== 没办法直接创建此目录啊！
# 加了这个 -p 的选项，可以自行帮你创建多层目录！
[root@VM-0-17-centos home]# mkdir -p  test1/test2

rmdir ( 删除空的目录 )

语法：

rmdir [-p] 目录名称

选项与参数：**-p ：**连同上一级『空的』目录也一起删除

测试：

# 看看有多少目录存在？
[root@VM-0-17-centos home]# ls -l
total 16
drwxr-xr-x 2 root root 4096 Feb 26 14:45 kevintam
drwxr-xr-x 2 root root 4096 Feb 26 14:51 test
drwxr-xr-x 3 root root 4096 Feb 26 14:51 test1
drwxr-xr-x 2 root root 4096 Feb 26 14:46 tom
# 可直接删除掉，没问题
[root@VM-0-17-centos home]# rmdir test
# 因为尚有内容，所以无法删除！
[root@VM-0-17-centos home]# rmdir test1/
rmdir: failed to remove ‘test1/’: Directory not empty
# 利用 -p 这个选项，立刻就可以将 test1/test2/test3/test4 依次删除。
[root@VM-0-17-centos home]# rmdir -p test1/test2/

注意：这个 rmdir 仅能删除空的目录，你可以使用 rm 命令来删除非空目录.

cp ( 复制文件或目录 )

语法：

[root@www ~]# cp [-adfilprsu] 来源档(source) 目标档(destination)
[root@www ~]# cp [options] source1 source2 source3 .... directory

选项与参数：

  • -a：相当於 -pdr 的意思，至於 pdr 请参考下列说明；(常用)
  • -p：连同文件的属性一起复制过去，而非使用默认属性(备份常用)；
  • -d：若来源档为连结档的属性(link file)，则复制连结档属性而非文件本身；
  • -r：递归持续复制，用於目录的复制行为；(常用)
  • -f：为强制(force)的意思，若目标文件已经存在且无法开启，则移除后再尝试一次；
  • -i：若目标档(destination)已经存在时，在覆盖时会先询问动作的进行(常用)
  • -l：进行硬式连结(hard link)的连结档创建，而非复制文件本身。
  • -s：复制成为符号连结档 (symbolic link)，亦即『捷径』文件；
  • -u：若 destination 比 source 旧才升级 destination ！

测试：

# 找一个有文件的目录，我这里找到 root目录
[root@kuangshen home]# cd /root
[root@kuangshen ~]# ls
install.sh
[root@kuangshen ~]# cd /home

# 复制 home目录下的hello 到 opt目录下
[root@VM-0-17-centos home]# cp -r hello /opt/
# 再次复制，加上-i参数，增加覆盖询问？
[root@VM-0-17-centos home]# cp -r hello /opt/
cp: overwrite ‘/opt/hello/hello.txt’? y # n不覆盖，y为覆盖
#不提示是否覆盖
\cp -r /root/install.sh /home

rm ( 移除文件或目录 )

语法：

rm [-fir] 文件或目录

选项与参数：

  • -f ：就是 force 的意思，忽略不存在的文件，不会出现警告信息；
  • -i ：互动模式，在删除前会询问使用者是否动作
  • -r ：递归删除啊！最常用在目录的删除了！这是非常危险的选项！！！

测试：

# 将刚刚在 cp 的实例中创建的 install.sh删除掉！
[root@VM-0-17-centos opt]# rm -rf hello
[root@VM-0-17-centos home]# rm hello.txt 
rm: remove regular empty file ‘hello.txt’? y
# 如果加上 -i 的选项就会主动询问喔，避免你删除到错误的档名！

# 尽量不要在服务器上使用 rm -rf /

mv ( 移动文件与目录，或修改名称 )

语法：

[root@www ~]# mv [-fiu] source destination
[root@www ~]# mv [options] source1 source2 source3 .... directory
#如果是在一个目录下的移动就是重命名

选项与参数：

  • -f ：force 强制的意思，如果目标文件已经存在，不会询问而直接覆盖；
  • -i ：若目标文件 (destination) 已经存在时，就会询问是否覆盖！
  • -u ：若目标文件已经存在，且 source 比较新，才会升级 (update)

测试：

# 复制一个文件到当前目录
[root@kuangshen home]# cp /root/install.sh /home

# 创建一个文件夹 test
[root@kuangshen home]# mkdir test

# 将复制过来的文件移动到我们创建的目录，并查看
[root@kuangshen home]# mv install.sh test
[root@kuangshen home]# ls
test
[root@kuangshen home]# cd test
[root@kuangshen test]# ls
install.sh

# 将文件夹重命名，然后再次查看！
[root@VM-0-17-centos home]# touch cat
[root@VM-0-17-centos home]# 
[root@VM-0-17-centos home]# mv cat pig
[root@VM-0-17-centos home]# ls
kevintam  pig  tom

创建一个空的文件

touch [文件名称]#指令

测试

[root@VM-0-17-centos kevintam]# touch hello.java
[root@VM-0-17-centos kevintam]# ls
hello.java

查看文件的内容

cat #查看文件的内容
-n #显示行号
#查看/etc/profile的文件 -n就是查看行号.
[root@VM-0-17-centos tom]# cat -n /etc/profile
# | more 管道命令 按回车键往下走 输入空格 会直接下一页
[root@VM-0-17-centos tom]# cat -n /etc/profile | more

more指令是一个vi编辑的文本过滤器，他以全屏的方式按页显示内容

more中常见的快捷键

  • 空格键: 往下翻一页
  • enter ：往下走一段
  • q：立刻离开
  • ctrl+f：向下滚动一屏
  • ctrl+b：返回上一屏
  • =：输出当前行号
  • ：f 输出当前的行号和文件名

cat和more通常组合使用

less 指令

less指令支持用来分屏查看文件内容。通常使用在大文件的查看

常用指令：

  • 空格键:向下翻到一页
  • q：离开less指令
  • /字符：查询字符
  • [pagedown]：向下翻动一页
  • [pageup]：向上翻动一页

echo

输出内容到控制台

echo[选项] [输出内容]

[root@VM-0-17-centos tom]# echo $HOSTNAME
VM-0-17-centos
[root@VM-0-17-centos tom]# echo "hello world~~"
hello world~~

head
#文件的开头
#看5行的代码
head -n 5
[root@VM-0-17-centos tom]# head -n 5 /etc/profile
# /etc/profile

# System wide environment and startup programs, for login setup
# Functions and aliases go in /etc/bashrc

tail  -f#文件尾的内容 实时的监控文件

>和>>指令
#输出重定向和追加
[root@VM-0-17-centos tom]# echo "hello world" > data.txt #会覆盖该文件里的数据
[root@VM-0-17-centos tom]# echo "java" >> data.txt #在原数据的基础上会追加内容

In指令

软链接也称为符号链接,类似于windows里的快捷方式,主要存放了链接其他文件的路径

基本语法

in -s 原文件或目录(功能描述：给原文件创建一个软链接) 连接到的目录

引用案列

案列1:在/home目录下创建一个软链接myroot,连接到/root目录

案列2:删除软链接myroot

in -s /root /myroot
rm /home/myroot #删除这个目录即可

history

查看已执行过历史命令,也可以执行历史命令

基本语法

history (功能描述:查看已经执行过历史命令)

history

案列1:显示所有的历史命令

history

案列2:显示最近使用过的10个指令

history 10

案列3：执行历史编号为5的指令
